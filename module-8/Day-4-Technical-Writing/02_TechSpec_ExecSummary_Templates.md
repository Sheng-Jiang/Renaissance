# Exercise: Tech Spec & Executive Summary Templates
**Time: 4 Hours**

## Part 1: Technical Specification (120 mins)
Write a formal tech spec for one of the following:
- A system improvement for OpenClaw (from your security audit or architecture analysis)
- Your capstone project's core technical architecture
- A feature enhancement for a Module 3–6 project

### Template:

#### Problem Statement
> Grounded in evidence. Why does this need to be solved? Who is affected and how?

#### Goals & Non-Goals
**Goals:**
- [ ] Goal 1
- [ ] Goal 2

**Non-Goals (explicitly out of scope):**
- [ ] Non-goal 1
- [ ] Non-goal 2

#### Technical Design
- Architecture diagram (components and interactions)
- Data models
- API contracts (endpoints, request/response shapes)

#### Alternatives Considered
| Option | Pros | Cons | Why Not |
|---|---|---|---|
| Option A | ... | ... | ... |
| Option B | ... | ... | ... |

#### Security Considerations
Reference OWASP Top 10 for LLM Applications or your Day 3 audit findings.

#### Testing Strategy
How you'll verify correctness (unit, integration, E2E, security testing).

#### Rollout Plan
How you'll deploy safely (feature flags, canary, staged rollout).

---

## Part 2: Executive Summary (60 mins)
Write a **1-page maximum** executive summary of the SAME project.

### Template:

> **Recommendation:** We should invest $[X] in [solution] to achieve [outcome].
>
> **Why Now:** [Business context, competitive pressure, user pain]
>
> **Business Impact:** [Quantified: revenue, cost savings, efficiency gains]
>
> **Investment:** [Team size, timeline, technology costs]
>
> **Top Risk:** [What could go wrong and how you'll mitigate it]
>
> **Timeline:** [Milestones and delivery date]

---

## Part 3: Peer Cross-Check (60 mins)
Exchange BOTH documents with a partner. Review:
- 🔧 Tech spec: "Is there enough detail for an engineer to build this?"
- 💰 Exec summary: "Would I fund this? Is anything too technical?"
- 🔗 Cross-check: "Are these clearly the same project?"
