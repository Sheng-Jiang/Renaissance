# Exercise: The User Testing Script
**Time: 2 Hours**

## Phase 1: Script Preparation (30 mins)
Write a script to guide your user testing sessions. *Do not give them a list of buttons to click.* Give them goals to achieve.

### Template:
1.  **The Welcome (2 mins):** "Hi, thanks for helping us. We are testing a very early prototype, not testing *you*. If something breaks or doesn't make sense, that's our fault, not yours. Please think out loud as you use this."
2.  **Context Questions (3 mins):** "Tell me about how you currently handle [Problem Area]."
3.  **Task 1 (5 mins):** "[Scenario: e.g., 'You just finished a 1-hour client meeting. Show me how you would capture the action items using this tool.']"
4.  **Task 2 (5 mins):** "[Scenario 2]"
5.  **Task 3 (5 mins):** "[Scenario 3]"
6.  **Wrap-Up (3 mins):** "What was the most frustrating part of this experience? If you had a magic wand, what one thing would you change?"

## Phase 2: Execution (60 mins)
Run 3 to 5 testing sessions. (Use external users, or if necessary, students from other hackathon teams who haven't seen your product).

*   **Observer:** Record *everything*. "User clicked the logo trying to go home. User hesitated for 10 seconds on the pricing page. User said 'Where do I save this?'"

## Phase 3: Synthesis & Action (30 mins)
1.  **Affinity Map:** Review the observer notes. Find the patterns. Did 3 out of 5 users fail to find the submit button?
2.  **Triage:** Sort findings into three buckets:
    *   **Fix Now:** Critical blockers. The core flow is broken.
    *   **Fix Tomorrow:** Huge UX improvements, but the app technically works.
    *   **Fix Later:** Nice-to-haves. (Ignore these for the rest of the hackathon).
3.  **Decision:** Update your GitHub Projects board with the "Fix Now" and "Fix Tomorrow" tickets. This is your backlog for Day 4.
