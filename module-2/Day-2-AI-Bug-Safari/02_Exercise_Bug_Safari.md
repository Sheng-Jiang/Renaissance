# Exercise: The AI Bug Safari (Rounds 1 & 2)

## Round 1: Planted Bugs (90 Minutes)
Your instructor will provide a repository containing 10 code samples. Each was generated by an AI and contains 1 to 3 distinct bugs mapable to the 6-part Failure Taxonomy.

**Instructions:**
1. Work individually for 45 minutes to spot the errors.
2. Pair up for 30 minutes to compare findings and debate.
3. We will reconvene to reveal the answer key. 

*Pay special attention to Security and Performance. They are the hardest to spot by eye.*

---

## Round 2: Wild Bugs (75 Minutes)
Now, hunt for bugs in the wild.

**Instructions:**
Use Claude Code CLI, Kiro, or Antigravity to generate code for the following four tasks:
1.  **Task A:** "Build a rate limiter middleware using a sliding window algorithm."
2.  **Task B:** "Implement a JWT refresh token rotation system."
3.  **Task C:** "Create a data migration script that transforms user profiles from schema v1 to v2."
4.  **Task D:** "Build a webhook receiver with retry logic and idempotency."

**Verification:**
For every generated solution, apply the **Verification Lens**. Document every bug, edge-case failure, or concern you find. Classify them using the taxonomy. 

*Goal: Find at least 5 real bugs from this round to add to your Bug Log.*
