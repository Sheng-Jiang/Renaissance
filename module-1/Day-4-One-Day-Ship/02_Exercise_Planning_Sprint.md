# Exercise: Architecture & Planning Sprint
**Time: 30 Minutes**

## Overview
The most common failure mode for a one-day ship is over-scoping. Before writing a single line of code, you must define the boundaries of your project. This exercises the **Precision** mindset.

## Instructions
Write a planning document in your repository (or NotebookLM) containing:

1.  **1-Paragraph Description:** What are you building and who does it serve?
2.  **Architecture Sketch:** List the frontend, backend, database, and any external APIs you will use. Keep it as simple as possible.
3.  **User Stories:** Define 3–5 user stories that define what "done" realistically looks like by 4:00 PM.
4.  **Tool Selection:** Which AI tool will you use for which part of the build? (e.g., Kiro for backend API, Claude for frontend).

## Setup Tracking
*   Create GitHub Issues for each of your user stories.
*   Add them to your GitHub Projects board.

## Instructor Review
Before you start coding at 10:00, an instructor will do a 2-minute scope review with you. *Expect them to tell you to cut your scope aggressively.*
