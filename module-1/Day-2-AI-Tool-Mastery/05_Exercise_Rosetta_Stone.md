# Exercise: The Rosetta Stone
**Time: 75 Minutes**

## Overview
This is the day’s *crown jewel*. It builds tool selection judgment. You will build the exact same feature using three different tools to understand their strengths and weaknesses.

## Task
**"Add a `/health` endpoint that returns system uptime, version, and dependency status."**

## Instructions

### Step 1: The Build
Create a new branch for each tool's attempt so you can compare them later.

1.  **Claude Code CLI:** Build the endpoint using a terminal-first workflow.
2.  **Kiro:** Build it using the spec-driven workflow (Requirements $\rightarrow$ Design $\rightarrow$ Code).
3.  **Antigravity:** Build it using the agent-assisted workflow.

*Ensure you commit each version on its corresponding separate branch!*

### Step 2: Analysis & Write-up
Write up your findings in your repository's wiki. Address the following:

1.  Which tool felt the fastest?
2.  Which tool produced the highest quality (or most robust) code?
3.  Which tool's output was the easiest to manually verify?
4.  What surprised you the most about the differences between them?
5.  **Most Importantly:** When would you choose each tool in a *real* project?

### Step 3: Class Discussion
Share your findings. There are no right or wrong answers, but there *are* wrong tools for the wrong situations.
