# Exercise: Antigravity Multi-Agent Orchestration
**Time: 60 Minutes**

## Overview
Antigravity uses multi-agent orchestration to accelerate development. You will experiment with its three development modes: Agent-Driven, Review-Driven, and Agent-Assisted.

## Pre-requisites
Install and connect Antigravity to GitHub. Ensure you are familiar with its UI and logging artifacts.

## Part 1: Review-Driven Mode
Start here for safety and verification practice.
**Task:**
> "Add a REST API endpoint with input validation and error handling."

**Action:**
In Review-Driven mode, Antigravity will pause and prompt you before executing critical steps. Review *every* step the agent proposes before clicking approve. Look at:
*   File creation/modification paths.
*   The exact bash commands it wants to run.
*   The logic it plans to inject.

## Part 2: Agent-Assisted Mode
Switch to Agent-Assisted (or Agent-Driven) mode and run a similar task (e.g., "Add unit tests for the previous endpoint"). Compare the experience.

## Part 3: Artifacts and Manager View
*   Explore the audit trail Antigravity leaves behind. Review its task lists, plans, screenshots (if applicable), and browser recordings.
*   **Preview:** The Manager View for orchestrating 5+ agents in parallel. We will use this heavily in Weeks 3 and 6, but get familiar with the interface now.
